# 安全说明

## 凭据

- 代理服务始终启用身份验证，禁止无密码公网开放。
- `deployment.local.json`、`deployment-*.local.json` 和 `extension/runtime-config.json` 已被 Git 忽略，不得手工提交。
- 扩展只在认证请求的主机和端口与已保存服务器完全一致时返回用户名和密码。
- 弹出窗口不会从后台读取并展示已保存的明文密码。
- 首次部署生成高强度随机代理密码；怀疑泄露时应重新生成并同步到使用该服务器的电脑。

代理密码保存在 `chrome.storage.local`。普通网页无法读取，但它不能抵御已经以当前 Windows 用户身份运行的恶意程序。

## TLS 与证书

公网代理使用 HTTPS，凭据和流量不会以明文发送。服务器使用 Let's Encrypt 的可信 IP 地址短期证书，并每天两次检查续期。安装器固定依赖版本并校验下载文件 SHA-256，避免无校验执行远程二进制。

## SSRF 与端口限制

认证通过不等于允许任意访问。回环安全出口会在域名解析后：

- 拒绝服务器自身、IPv4/IPv6 私网、回环及链路本地目标；
- 只允许目标端口 80 和 443；
- 拒绝其他所有目标。

这可以降低代理被用于扫描服务器内网、访问云元数据或连接非 Web 服务的风险。

## systemd 隔离

GOST 和 sing-box 均以无登录权限的 `browser-gateway` 用户运行，并启用 `NoNewPrivileges`、只读系统目录、私有临时目录、能力白名单、地址族限制和内存上限。只有 GOST 获得绑定 443 所需的 `CAP_NET_BIND_SERVICE`。

## SSH

服务器部署完成并验证密钥登录后，建议禁用 SSH 密码登录，只允许密钥。不要删除云服务商的控制台救援入口。SSH 私钥不得上传仓库或复制到不受信任设备。

## 验收

`tools/test-server.ps1` 会验证：

- 正确凭据可以访问公网，且出口 IP 正确；
- 错误凭据被拒绝；
- 私网目标被拒绝；
- 非 80/443 目标端口被拒绝。

`tools/test-chrome-proxy.mjs` 进一步使用真实 Chromium 验证扩展控制、认证、GitHub/ChatGPT 访问及 HTTP/2 代理会话。
